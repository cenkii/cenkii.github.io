2、Linux中的目录

bin：常用指令命令

sbin：高权限用户命令，管理员使用的命令

home：普通用户对应文件，即主目录，目录名是以用户名命名的

root：系统管理员对应的主目录

etc：存放所有系统管理所需的配置文件

usr/local：用户安装的文件，应用程序

boot：启动Linux时使用的一些核心文件，包括链接文件以及镜像文件

media：识别dvd，U盘

mnt：让用户临时挂载别的文件系统，外部存储挂在在这个文件夹下

var：变量日志

======================================================================

opt：一般存放安装软件程序

tmp：临时文件

dev：将硬件设备映射为文件管理

lib：，lib64：库文件，动态连接共享库

=========================大佬文件别轻易动================================

proc：内核文件，系统内存的映射访问他来获取系统信息

sys：系统2.6之后出现出现的文件系统

srv：存放了一些服务启动需要提取的数据

=======================================================================

2）只有一个根目录

3）文件要放到对应位置

1）每个用户必须要属于一个组，不指定会默认创建

inittab is no longer used when using systemd.

ADDING CONFIGURATION HERE WILL HAVE NO EFFECT ON YOUR SYSTEM.

Ctrl-Alt-Delete is handled by /usr/lib/systemd/system/ctrl-alt-del.target

systemd uses ‘targets’ instead of runlevels. By default, there are two main targets:

multi-user.target: analogous to runlevel 3

graphical.target: analogous to runlevel 5

To view current default target, run:

systemctl get-default

To set a default target, run:

systemctl set-default TARGET.target

6、操作指令

获取帮助

man 命令或配置

help 命令

文件目录类指令

pwd

pwd（print work directory）显示当前目录绝对路径

ls ( list files ) 指令

  • ls [选项] [目录或文件]
  • ls -l 显示详细信息 包括权限所有者
  • ls -al 显示所有文件包括隐藏文件

  • cd ~ 回到家目录

3、权限

drwxr-xr-x. 2 root root 6 4月 30 20:18 桌面

​ 权限 数字 所有者 所有者组 大小 文件最后修改时间 文件名

第一位：代表文件类型

​ d ：文件夹

​ - ：文件

​ l ：软链接

​ c ：字符设备（鼠标键盘）

​ b：块文件（硬盘）

第二三四位：代表所有者权限

​ rwx：代表有读写（进入目录或执行文件）权限

第五六七位：文件所在组用户拥有权限

​ r-x：代表有读（进入目录或执行文件）权限，没写权限

第八九十位：其他组用户拥有的权限

​ r-x：代表有读（进入目录或执行文件）权限，没写权限

数字：

​ 文件1：代表硬链接数

​ 目录2：代表子目录个数

大小：

​ 文件显示文件所占字节

​ 目录显示6

权限修改

chomd

u：所有者 g：所在组 o：其他组 a：所有人

+：增加

-：去除

1、chmod u - w 文件名 （去掉所有者写权限）

2、chmod o + w 文件名 （为所其他人增加写权限）

3、chmod u=rwx,g=rx,o=x 文件名

方式二

r = 4 ， w = 2 ， x = 1

chmod 777 文件名 （所有用户对文件开启所有权限）

修改所有者

chown owner filename

chown owner:group filename （同时修改拥有者及组）

chown -R owner dirname （修改所在目录所有文件）

修改所在组

chgrp groupname file

chgrp -R groupname dir （将dir下全修改）

上述 -R 中的R必须大写

8、crond任务调度

crontab [选项]

常用选项

-e 编辑

-l 显示

-r 删除所有

service crond restart （重启任务调度服务）

cron占位符使用说明

与spring中的cron表达式一样

项目 含义 范围
第一颗 “*” 代表一小时的第几分钟 0-59
第二颗 “*” 代表一天中的第几小时 0-23
第三颗 “*” 代表一个月的第几天 1-31
第四颗 “*” 代表一年中的第几月 1-12
第五颗 “*” 代表星期 0-7（07含义相同）
* 代表任何时间
， 分隔离散时间 1，2，3
- 代表范围
*/n 每个多久执行一次 “*/10 * * * *” 10分钟一次

使用脚本流程

1）编写脚本文件

例如：

vim /home/mytask.sh

将date信息追加写入/tmp/mydate文件

date >> /tmp/mydate

2）给mydate.sh可执行权限

chmod 744 mydate.sh

3）crontab -e

每隔1min执行一次

*/1 * * * * /home/mytask.sh

常用分区

特点

1.最多四个主分区

2.系统只能安装在主分区

3.扩展分分区占一个主分区

4.最大值支持2TB兼容性好

gtp分区

特点

1.支持无限主分区

11、进程管理

基本介绍

1）每执行一个程序就会启动一个进程。每一个进程对应着一个进程号。

2）每一个进程对应着一个父进程，这个父进程可以对应多个子进程

3）进程可以以前台和后台两种形式存在，一般系统服务会以后台方式运行，直到关机才结束进程

查看进程指令

-a 显示所有进程信息

-u 以用户格式显示进程信息

-x 显示后台进程运行参数

含义（第一行为例）

root 用户名（USER）

1 进程id（PID）

0.0 cpu使用情况（%CPU）

0.2 内存使用情况（%MEM）

193912 使用的虚拟内存（VSZ）

4580 使用的物理内存（RSS）

？ 使用的终端（TTY）

Ss 进程状态（STAT s：休眠 r：运行 z：僵尸进程）

启动时间（START）

占用cpu总时间（TIME）

执行时命令（COMMAND）

以全格式显示所有进程，查看父进程 PPID即为父进程id

终止进程

kill和killall
